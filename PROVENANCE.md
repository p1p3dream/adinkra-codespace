# Provenance: 10D Supergravity L & R Dataset (`data/tendim_10d_lr.json`)

This document records the exact origin, regeneration procedure, conventions, and
licensing posture of the 10D L/R matrix dataset shipped in this repository.

It deliberately does **not** claim the dataset is byte-faithful to any matrices
published by the upstream authors. What it claims is precise and verifiable: the
dataset is an academic **regeneration** produced by numerically evaluating the
authors' published generative Mathematica ("Garden Algebra") at a pinned commit,
with the conventions and known discrepancies documented below.

---

### Canonical content hash (the integrity check that matters)

The raw-byte SHA256 of the JSON is **not** the canonical integrity check, because:

1. Adding/editing metadata fields changes the raw bytes without changing any matrix value.
2. Floating-point formatting and key ordering can drift across serializers.

The canonical check is the **symbolic-token content hash** computed by
`canonical_token_hash()` in `src/tendim_data.rs`. It snaps every L/R entry to its
exact symbolic token (e.g. `-2`, `1/16`, `1/sqrt8`), frames the tokens in a fixed
structural order (matrix index, row, col) under the prefix `TENDIM-TOKENS-v1;`,
and SHA-256s that stream. This hash depends **only** on matrix content, not on
JSON byte layout, so it is stable across reserialization and metadata edits. Use
`canonical_token_hash()` (not the raw-byte SHA256) to verify dataset content.

## 3. Regeneration

The dataset is reproduced deterministically from the pinned upstream source by
running the regeneration script from the repository root:

```sh
python3 scripts/gen_10d_data.py
```

The companion paper (arXiv:2512.12157) was consulted locally; its text was
extracted with `pdftotext` (`pdftotext` output cached at `/tmp/2512_pdf.txt`) to
cross-check degree-of-freedom counts and the worked examples referenced below.

## 4. Conventions

These conventions define the basis in which the matrices are expressed. Two
regenerations agree only if they share these conventions; a different (but
physically equivalent) basis would produce a permuted/sign-flipped dataset.

### 4.1 Colors (generators)

| Convention | Value |
|------------|-------|
| Color index `I` | runs `1 .. 16` |
| Number of generators `n` | 16 |

### 4.2 Boson ordering (`nb = 82`)

| Block | Count | Description |
|-------|-------|-------------|
| Graviton h-pairs (symmetric) | 45 | symmetric `h` components, `{1,1}` pair first |
| `B` field pairs (antisymmetric) | 36 | antisymmetric `B` components |
| Dilaton `Phi` | 1 | single scalar |
| **Total** | **82** | |

DOF derivation (paper Eq. 6.0.3, arXiv:2512.12157, text lines 763-774):
`nb = dof(h_{mu nu}) + dof(B_{mu nu}) + dof(phi) = 55 + 45 + 1 - 19 = 82`,
where the `-19` is the gauge fixing (`h_{0 nu}=0`: 10 components, `B_{0 nu}`: 9 components).

### 4.3 Fermion ordering (`nf = 176`)

| Block | Count | Description |
|-------|-------|-------------|
| Gravitino `psi` | 160 | `psi_{mu, dot a}` |
| Dilatino `chi` | 16 | `chi_a` |
| **Total** | **176** | |

DOF derivation (paper Eq. 6.0.4): `nf = dof(psi_{mu, dot a}) + dof(chi_a) = 160 + 16 = 176`.

### 4.4 Matrix shapes

`L_I` is `nb x nf` (82 x 176); `R_I` is `nf x nb` (176 x 82); 16 of each.

### 4.5 Metric signature

`SO(1,9)` mostly-plus signature `(-,+,+,+,+,+,+,+,+,+)` (one timelike, nine spacelike).

### 4.6 Normalization constants

The Garden Algebra evaluation introduces the following normalization factors:

| Constant | Where it appears |
|----------|------------------|
| `1/2` | sigma-bilinear (`Sig2`/`TildeSig2`) antisymmetrizations |
| `1/8` | R-side entries (token `1/8`) |
| `1/16` | R-side entries (token `1/16`) |
| `1/sqrt8` | dilaton (`Phi`) coupling on the L side; R-side token `1/sqrt8` |
| `sqrt8` | dilatino (`chi`) coupling on the L side (token `sqrt8`) |

### 4.7 Allowed entry value-sets

| Matrix | Allowed exact tokens |
|--------|----------------------|
| `L` | `{ -2, -1, 0, 1, 2, sqrt8 }` (both signs of `sqrt8` admitted defensively; dataset uses `+sqrt8`) |
| `R` | `{ 0, +-1/16, +-1/8, +-7/16, +-1/2, +-1/sqrt8 }` |

### 4.8 Defining relations

| Side | Relation |
|------|----------|
| Bosonic | `L_I R_J + L_J R_I = 2 delta_IJ I_82` |
| Fermionic | `R_I L_J + R_J L_I = 2 delta_IJ I_176 + 2 E_IJ` |

The bosonic Garden relation holds to ~1.7e-12 (computed here). The fermionic
`E_IJ` remnant is **computed here**, not quoted from the paper.

## 5. License and citation posture

The upstream repository `github.com/mcmulaz/Super-Sym` contains **no LICENSE
file** at commit `8c8df92`. There is therefore no granted redistribution license
for any upstream artifact.

This repository does **not** redistribute a licensed upstream dataset. Instead it
ships an **academic regeneration**: the L/R matrices here are produced by
numerically evaluating the authors' own published generative Mathematica
("Garden Algebra"), and the result is provided for scholarly reproducibility with
full attribution to the source and the companion paper.

Anyone using this dataset should cite the source paper:

> arXiv:2512.12157, "10D Supergravity Numerical Data Sets for L & R Matrices",
> with generative code at `github.com/mcmulaz/Super-Sym` (commit `8c8df92`).

If the upstream authors add a license or object to redistribution, this dataset
posture should be revisited.

## 6. Paper worked-example discrepancy (UNRESOLVED)

The paper's **displayed worked examples** disagree with the output of the upstream
Garden Algebra (and therefore our regeneration). This was investigated with two
adversarial passes; it is **not resolved** and an earlier "it's a typo" conclusion
was retracted after review.

What disagrees:
- Eq. 6.0.5 bracketed spinor indices: e.g. paper `Q_1 h_11 = 2 psi_1(6)` vs the
  Garden source / our JSON `2 psi_1(16)`; the discrepancy is across the displayed
  examples and not a single fixed permutation.
- Eq. 6.0.6 gravitino (psi) examples ALSO disagree on coefficients: e.g. paper
  `Q_1 psi_1(1) = i/2 d h19 + i/16 d B19` vs our R row `+1/2 h19 - 7/16 B19`. The
  chi examples and the phi line do match.

What is established (and IS solid):
- Our JSON **faithfully reproduces the authors' Garden Algebra source** at commit
  `8c8df92`: an exact (sympy) re-derivation from that source matches our JSON
  entrywise (0 mismatches) and the bosonic Garden relation holds exactly
  (`scripts/eval_garden_exact.py`).

What is NOT established:
- Whether the Garden source equals the paper's *intended* matrices, or whether the
  paper's displayed examples contain transcription errors, or whether the
  difference is a primitive-vs-assembled coefficient presentation (e.g. the paper's
  `i/16` vs our assembled `7/16`). A reconciling convention map has not been proven,
  and our own automated checks disagreed on the Eq. 6.0.6 match.

**Status: unresolved; requires confirmation from the authors.** We make NO typo
claim and NO paper-fidelity claim. The defensible statement is only: "this dataset
reproduces the authors' Garden Algebra generative source; the paper's displayed
worked examples disagree with that source and the discrepancy is unresolved."

## 7. What is and is not claimed

- **Claimed:** the dataset is an exact numerical regeneration of the upstream
  "Garden Algebra" code at the pinned commit; the bosonic Garden relation holds
  to ~1.7e-12; all entries snap to the documented exact token value-sets; the DOF
  counts match the paper (Eq. 6.0.3-6.0.4).
- **Not claimed:** that the dataset is byte-faithful to any matrices the authors
  published, or that it reproduces every worked example in the paper (see the
  known mismatch in section 6).
